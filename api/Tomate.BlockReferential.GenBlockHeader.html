Remarks

MemoryBlock instances must be free-able without specifying the allocator that owns it and also supporting thread-safe lifetime related operations, so this header is the way to ensure all of this. More specifically, IBlockAllocator is the interface that takes care of freeing a MemoryBlock and the way to identify which particular instance of the Memory Manager is through the BlockIndex property. Be sure to check out DefaultMemoryManager to get and understand how things are working.
